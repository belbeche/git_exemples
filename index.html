Exemple d'utilisation de Git

  • git init : permet d'iniliaser un repository git à la racine du dossier
  • git status : permet de traquer les changements liées aux états
  • git add : pour ajouter des fichiers en attente d'être sauvegardé
  • git commit -m "...." : l'intitulé de la sauvegarde
  • git commit -am "..." : on ajoute les fichiers et on les sauvegarde
  • git log : affiche la liste des commits (=sauvegardes)
  • git diff : permet de voir les dernières modifcations
  • git reset HEAD : enregistre un etat permettant de revenir dessus

    Git reset HEAD est une commande Git qui permet de retirer des fichiers de l'index (ou staging area) pour annuler un commit précédent ou pour désindexer des modifications qui ont été préparées pour un commit. L'index est l'espace de travail intermédiaire entre le répertoire de travail (le répertoire où l'ont travaille sur des fichiers) et l'historique de commit (la série de commits qui ont été enregistrés dans le dépôt Git).

  • git reset --hard : on force la remise à zero, on revient à l'état du commit HEAD en supprimant le code
  • git reset ~1 : on revient sur l'état précedent du dernier commit sans supprimer le code
  • git restore --staged *.html : on enlève les fichiers en attente d'être sauvegarder
